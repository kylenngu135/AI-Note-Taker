## Project Overview

AI Note Taker — uploads documents (PDF, DOCX, TXT) or audio/video files, transcribes content, and generates AI-powered study notes via Groq's Llama 3.3 LLM. Cloudflare R2 stores transcriptions and notes; PostgreSQL stores metadata.

## Common Commands

```bash
# Start all services (PostgreSQL, Whisper, Go backend)
docker-compose up --build

# Run backend tests
cd src/backend && go test ./...

# Build Go backend locally (requires PostgreSQL, R2, and Groq credentials in .env)
cd src/backend && go build -o main .
```
Note: The Go backend also serves the frontend static files from `src/ui/` via its Dockerfile — the Go binary embeds/paths to the ui directory.

## Architecture

```
Browser → Go Backend (:8080) → PostgreSQL (metadata)
                  ↓           R2 (files/blobs)
                  ↓
         Python Whisper (:8081) → audio/video transcription
                  ↓
           Groq API (Llama 3.3) → study sheet generation
```

### Backend (Go, src/backend/)
- `main.go` — HTTP server entry point, registers all routes
- `api/uploads.go` — handles file upload, retrieval, deletion; routes: `POST/GET/DELETE /api/uploads/*`
- `api/databases.go` — PostgreSQL queries for uploads, notes, users
- `auth/auth.go` — register, login, logout, JWT generation
- `middleware/enable_cors.go` — CORS headers
- `middleware/validate.go` — JWT validation middleware
- `notes/generator.go` — calls Groq API to generate study sheets
- `storage/r2.go` — Cloudflare R2 S3-compatible client (AWS SDK v2)
- `transcription/document.go` — PDF (`ledongthuc/pdf`), DOCX (`fumiama/go-docx`), TXT text extraction
- `transcription/whisper.go` — calls Python Whisper service over HTTP
- `migrations/` — golang-migrate SQL migrations (PostgreSQL)
- `whisper/` — Python Flask service with OpenAI Whisper model for audio/video transcription

### Frontend (src/ui/ and src/frontend/)
- `src/ui/index.html` — main SPA interface
- `src/ui/auth.html` — authentication page
- `src/ui/styles.css` — styling
- `src/frontend/main.js` — entry point, imports submodules
- `src/frontend/FileUpload.js`, `DisplayRows.js`, `delete.js`, `notes.js`, `auth.js`, `ui.js` — modular JS

### Docker Services
- `backend` — Go server, port 8080; also serves frontend static files
- `whisper` — Python Flask + OpenAI Whisper, port 8081
- `db` — PostgreSQL 16, port 5432

## Key Implementation Notes

- The Go backend serves both the API and static frontend files (Dockerfile copies `src/ui/` into the container)
- Document transcription (PDF/DOCX/TXT) is done entirely in Go; audio/video is forwarded to the Python Whisper service
- Study notes are generated by sending extracted text to Groq's `llama-3.3-70b-versatile` model
- Auth uses JWT (HS256) with bcrypt password hashing; tokens stored client-side
- R2 stores raw transcription `.txt` files and generated study sheet `.txt` files, keyed by upload ID

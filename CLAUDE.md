## Project Overview

AI Note Taker ‚Äî uploads documents (PDF, DOCX, TXT) or audio/video files, transcribes content, and generates AI-powered study notes via OpenAI GPT-4o-mini. Cloudflare R2 stores transcriptions and notes; PostgreSQL stores metadata.

## Common Commands

```bash
# Run the Go backend (requires PostgreSQL, R2, and OpenAI credentials in .env)
cd src/backend && go run .

# Run backend tests
cd src/backend && go test ./...

# Build Go backend binary
cd src/backend && go build -o main .
```

## Architecture

```
Browser ‚Üí Go Backend (:8080) ‚Üí PostgreSQL (metadata)
                  ‚Üì           R2 (files/blobs)
                  ‚Üì
         OpenAI Whisper API ‚Üí audio/video transcription
                  ‚Üì
           OpenAI GPT-4o-mini ‚Üí study sheet generation
```

### Backend (Go, src/backend/)
- `main.go` ‚Äî HTTP server entry point, registers all routes
- `api/uploads.go` ‚Äî handles file upload, retrieval, deletion; routes: `POST/GET/DELETE /api/uploads/*`
- `api/databases.go` ‚Äî PostgreSQL queries for uploads, notes, users
- `auth/auth.go` ‚Äî register, login, logout, JWT generation
- `middleware/enable_cors.go` ‚Äî CORS headers
- `middleware/validate.go` ‚Äî JWT validation middleware
- `notes/generator.go` ‚Äî calls OpenAI GPT-4o-mini to generate study sheets
- `storage/r2.go` ‚Äî Cloudflare R2 S3-compatible client (AWS SDK v2)
- `transcription/document.go` ‚Äî PDF (`ledongthuc/pdf`), DOCX (`fumiama/go-docx`), TXT text extraction
- `transcription/whisper.go` ‚Äî calls OpenAI Whisper API (`/v1/audio/transcriptions`) for audio/video; uses `OPENAI_API_KEY`
- `migrations/` ‚Äî golang-migrate SQL migrations (PostgreSQL)

### Frontend (src/ui/ and src/frontend/)
- `src/ui/index.html` ‚Äî main SPA interface
- `src/ui/auth.html` ‚Äî authentication page
- `src/ui/styles.css` ‚Äî styling
- `src/frontend/main.js` ‚Äî entry point, imports submodules
- `src/frontend/FileUpload.js`, `DisplayRows.js`, `delete.js`, `notes.js`, `auth.js`, `ui.js` ‚Äî modular JS

## Key Implementation Notes

- The Go backend serves both the API and static frontend files from `src/ui/`
- Document transcription (PDF/DOCX/TXT) is done entirely in Go; audio/video is sent to the OpenAI Whisper API
- Study notes are generated by sending extracted text to OpenAI's `gpt-4o-mini` model
- Auth uses JWT (HS256) with bcrypt password hashing; tokens stored client-side
- R2 stores raw transcription `.txt` files and generated study sheet `.txt` files, keyed by upload ID
